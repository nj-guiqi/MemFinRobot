### 2.1.2 智能体架构的发展

如果说大语言模型（LLMs）为多轮对话提供了强大的语言理解与生成能力，那么“智能体（Agent）架构”的研究则回答了另一个更关键的问题：**如何将语言能力组织成可执行、可控、可扩展、可评测的系统能力**。尤其在证券投资咨询这类高风险、强监管场景中，系统不仅要“答得像”，更要在长链路交互中做到**记忆可控、工具可用、输出可审计、表达边界可稳定约束**。本文所面向的证券咨询长期陪伴式交互痛点——用户画像难沉淀、跨轮约束易遗忘、风险披露口径漂移——本质上都指向“仅靠模型生成”难以满足的系统级稳定性问题，需要通过架构层面的模块化与闭环控制来解决。

#### （一）从“端到端对话”到“推理—行动”闭环：智能体作为系统而非单模型

早期对话系统多将“输入—输出”视为一次性生成过程，模型主要承担语言生成与知识拟合功能。然而，随着任务从开放域聊天走向复杂决策支持与真实世界执行（如检索、计算、调用业务接口），仅依赖端到端生成会面临两类结构性瓶颈：其一是事实与计算能力不足导致的幻觉与不可靠；其二是复杂任务需要多步分解与动态纠错，单次生成难以保证过程正确性。基于此，“推理与行动耦合”的智能体范式逐步兴起，其中 ReAct 框架具有代表性：它让模型在生成过程中交替输出**推理轨迹**与**外部行动（如检索/调用API）**，从而在执行反馈中迭代修正推理，并提升过程可解释性与可控性。([arxiv.gg][1])
这一范式的关键意义在于：多轮对话不再被视为“连续文本生成”，而是被重构为一个**面向目标的决策过程**，其核心能力来源于“可回溯的推理链 + 可验证的外部行动 + 可反馈的闭环更新”。

面向实际系统落地，智能体架构通常需要一个“控制层”将 LLM 的生成能力嵌入工作流：包括意图识别、任务分解、工具选择、结果整合与最终生成。本文在系统实现部分也明确给出了闭环运行流程：用户输入后，经由对话管理理解与分类，记忆模块提供支持，智能体以 ReAct 多步推理方式调用工具完成目标，最终由合规模块补足风险提示并进行表达约束，随后更新记忆形成持续迭代的交互闭环。
这一流程与 ReAct 的核心思想在自由发挥转化为可工程化的任务执行链条。

#### （二）从“单体模型”到“系统化模块”：MRKL 与“能力解耦”的架构观

随着 LLM 能力边界被系统性揭示，研究者逐渐形成共识：**复杂智能并非仅靠更大模型即可获得**，而需要引入系统工程视角，将不同能力以模块方式组合，并通过接口化机制实现协作。MRKL Systems 提出了一种典型的“系统化智能”理念：将语言模型作为中枢，但把知识获取、离散推理、计算与工具调用等能力交由外部模块实现，通过模块化组合规避纯语言模型在知识、推理与可靠性方面的固有限制。([UPDF AI][2])
这种架构观对强监管金融场景尤为关键：将“记忆”“工具”“合规”等能力从模型中解耦出来，能够把不可控的隐式行为转化为可审计、可测试、可迭代的显式模块。本文在方法对比与系统设计中也采取了类似的层次化技术路线：从纯 LLM（无记忆、无工具、无显式读写）到记忆增强模块（如独立长期记忆 mem0）、再到工具化记忆编排智能体（LangMem），最后发展为耦合记忆管理器、金融工具链与合规管理器的完整金融智能体架构。
这一演进路径体现了智能体架构的一个核心趋势：**从“能力堆叠”走向“能力编排（orchestration）”，并以模块解耦保证系统可扩展与可控。**

#### （三）从“能用工具”到“会学会用工具”：工具学习与执行可靠性增强

在工具增强路线上，Toolformer 进一步推动了“工具调用”从工程规则走向学习机制：模型通过少量示例与自监督方式学习何时调用什么工具、如检索等模型薄弱能力，同时尽量保持语言建模能力不受损。([arxiv.gg][3])
与手工编排相比，工具学习的意义在于使智能体具备一定的“可迁移调用能力”：当任务和工具集合扩展时，系统不必完全依赖人工规则调整。但在金融咨询场景，工具学习并不能替代合规与风险控制：工具输出的“事实正确”并不等价于“表达合规”，因此工具链应被视为提升证据充分性与可引用性的手段，而**输出边界仍需独立的合规模块进行过程约束**。这也对应本文提出的“记忆—工具—合规”三者协同架构：工具链提供行情查询、产品检索、计算与网页访问等能力以增强事实支撑，而合规管理器负责风险提示生成与禁区表达控制，从结构上保证输出可审计与可落地。

#### （四）从“短期上下文”到“长期认知状态”：记忆、反思与自我纠错机制

多轮对话智能体要实现长期陪伴式交互，仅靠上下文拼接难以维持稳定一致性，因而“显式记忆”逐渐成为主流架构组件，并进一步演化出“反思—记忆—再规划”的闭环。Reflexion 提出用语言化反馈驱动的自我反思机制：智能体将失败原因与改进策略写入记忆缓冲区，在后续尝试中以“可读文本记忆”而非参数更新的方式实现试错改进，从而提升连续任务执行能力。([arxiv.gg][4])Generative Agents 则提出“记忆流—反思—规划”的三段式架构，将经历以自然语言持续记录，通过反思生成更高层抽象的稳定认知，再在规划阶段动态检索相关记忆以驱动后续行为，从而支持更长时间跨度的一致行为生成。([prereview.org][5])

上述工作共同指向智能体架构的一个关键跃迁：记忆不再只是“召回相似文本”，而是被用来维护**可持续更新的认知状态**，并支持纠错与计划更新。本文也以“短期优先、长期补充、画像约束”的原则对多类型记忆进行编排：短期记忆保障语义连续，长期记忆提供跨时间背景，用户画像以结构化形式作为风险与偏好约束贯穿生成全过程。
该类分层记忆设计与反思型智能体的思想一致，即通过记忆模块将“长期一致性”从模型隐式行为转化为可工程化、可更新、可归因的系统能力。

#### （五）从“文本世界”到“真实世界”：具身智能体与可执行性约束

当智能体从文本交互走向对环境的真实操作，架构面临的核心挑战是“计划的可执行性”。SayCan 提出将语言模型的高层动作提议与低层技能的可供性/价值评估结合，用技能价值函数对 LLM 提议动作进行约束，从而让生成计划与真实环境能力对齐，避免“说得对但做不到”。([arxiv.gg][6])
Voyager 则展示了更长期的开放式具身学习：通过自动课程生成、可复用技能库与迭代式提示机制，将环境反馈、执行错误与自我验证纳入循环，在不微调模型参数([scixplorer.org][7])
尽管证券咨询不属于物理具身，但其“可执行性约束”同样存在：合规规则、风险披露清单、适当性要求等构成了“制度环境”的约束条件。类比而言，金融智能体也需要把“可执行/可发布”作为显式约束引入生成过程，而不是让模型自由生成后再被动筛查。本文的合规管理模块正是对这一思想的领域化实现：在生成阶段补足风险提示并抑制禁区表达，从架构层面降低跨轮漂移带来的合规风险。

#### （六）综述视角下的统一框架：Profiling–Memory–Planning–Action

为系统化刻画智能体架构的构成要素，LLM Autonomous Agents 相关综述工作提出了较为统一的分析框架：通常将智能体拆解为**画像/角色设定（profiling）、记忆（memory）、规划（planning）与行动（action/tool use）**等模块，并讨论其在不同应用中的组合形态与评测策略。([scixplorer.org][8])
结合证券投资咨询场景的工程需求，上述框架进一步强调了金融领域的特殊约束：一方面需要“画像”以结构化形式承载风险等级、期限偏好与禁忌偏好等关键状态，另一方面需要“行动/工具”提供事实证据与可引用依据，同时必须引入独立的“合规模块”对输出进行过程约束与可审计化封装。本文据此将长期一致性输出拆解为“记忆增强、智能体推理闭环与合规管理”三类可工程化能力，并以接口化方式实现模块解耦，便于扩展与真实业务接

#### （七）小结：智能体架构演进的主线是“可控系统化”

综上，智能体架构的发展呈现出清晰的主线：从端到端生成走向“推理—行动”闭环（ReAct），从单一模型走向模块化系统（MRKL），从工具可用走向工具可学习（Toolformer），从短期上下文走向长期认知状态与反思纠错（Reflexion、Generative Agents），并在具身与开放式环境中引入可执行性与反馈学习机制（SayCan、Voyager）。这些工作共同推动了一个结论：**多轮对话智能体的核心竞争力不再是“生成能力”，而是“状态管理 + 工具执行 + 过程约束”的系统能力**。这也为本文后续提出面向证券咨询的“记忆—工具—合规”协同架构提供了直接理论与工程脉络支撑：通过分层记忆与用户画像维护长期一致性，通过工具链增强事实支撑与可引用性，通过合规模块实现风险披露与表达边界的稳定控制，从而将长期陪伴式投顾从“模型能力展示”推进到“可评测—可迭代—可交付”的工程闭环。
