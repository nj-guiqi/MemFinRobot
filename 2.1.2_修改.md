## 2.1.2 智能体架构的发展（修改稿，含引用）

如果说大语言模型（LLMs）为多轮对话提供了强大的语言理解与生成能力，那么“智能体（agent）架构”的研究回答的是更关键的系统问题：**如何将语言能力组织为可执行、可控、可扩展、可评测的系统能力**。在证券投资咨询这类高风险、强监管场景中，系统不仅要“答得像”，更要在长链路交互中实现**记忆可控、工具可用、输出可审计、表达边界可稳定约束**。从工程视角看，长期陪伴式证券咨询的典型痛点——用户画像难沉淀、跨轮约束易遗忘、风险披露口径漂移——本质上都指向“仅靠模型生成”难以满足的**系统级稳定性**问题，因此需要通过架构层面的模块化与闭环控制加以解决（Wang 等，2023）。([arXiv][1])

### （一）从“端到端对话”到“推理—行动”闭环：智能体作为系统而非单模型

早期对话系统多将交互简化为一次性“输入—输出”生成，模型主要承担语言生成与知识拟合。然而，当任务从开放域聊天走向复杂决策支持与真实世界执行（检索、计算、调用业务接口等）时，端到端生成面临两类结构性瓶颈：一是事实/计算能力不足引发幻觉与不可靠；二是复杂任务需要多步分解与动态纠错，单次生成难以保证过程正确性。由此，“推理与行动耦合”的智能体范式逐步形成。Yao 等（2022）提出 ReAct 框架，使模型在生成过程中交替输出**推理轨迹**与**外部行动（如检索、API 调用）**，并利用执行反馈迭代修正推理，从而提升可解释性与可控性（Yao 等，2022）。([arXiv][2])

该范式的关键意义在于：多轮对话不再仅被视为“连续文本生成”，而被重构为**面向目标的决策与执行过程**，其系统能力来源于“可回溯推理链 + 可验证外部行动 + 反馈驱动的闭环更新”（Yao 等，2022）。([arXiv][2])
面向实际落地，智能体架构通常还需要“控制层”将 LLM 嵌入工作流，包括意图识别、任务分解、工具选择、结果整合与最终生成等环节（Wang 等，2023）。([arXiv][1]) 在你的研究语境中，可将流程表述为：用户输入→对话管理理解与分类→记忆模块提供状态/画像→智能体基于 ReAct 多步推理调用工具→合规模块补足风险提示并施加表达约束→更新记忆形成持续迭代闭环，从而将“自由生成”转化为可工程化的任务执行链条。

### （二）从“单体模型”到“系统化模块”：MRKL 与“能力解耦”的架构观

随着 LLM 能力边界被系统性揭示，研究者逐渐形成共识：**复杂智能并非仅靠更大模型自然获得**，而需要系统工程视角，通过模块化组合与接口协作实现能力互补。Karpas 等（2022）提出 MRKL Systems，将语言模型作为中枢，同时将知识获取、离散推理、计算与工具调用等交由外部模块承担，以模块化方式规避纯语言模型在知识、推理与可靠性方面的固有限制（Karpas 等，2022）。([arXiv][3])

该“能力解耦”观对强监管金融场景尤为关键：把“记忆”“工具”“合规”等能力从模型隐式行为中拆出，可将不可控的生成行为转化为**可审计、可测试、可迭代**的显式模块（Karpas 等，2022；Wang 等，2023）。([arXiv][3]) 因而在论文叙述中，你提出的从纯 LLM → 记忆增强 → 工具化记忆编排 → 记忆管理器+金融工具链+合规管理器的演进，可被概括为智能体架构的一条主趋势：**从“能力堆叠”走向“能力编排（orchestration）”，并以模块解耦保证系统可扩展与可控**。

### （三）从“能用工具”到“会学会用工具”：工具学习与执行可靠性增强

在工具增强路径上，Schick 等（2023）提出 Toolformer，使模型通过少量示例与自监督信号学习**何时调用何种工具、如何构造参数以及如何融合返回结果**，从而把“工具调用”从手工规则推进到可学习机制（Schick 等，2023）。([arXiv][4]) 相较于人工编排，工具学习的意义在于提升智能体的“可迁移调用能力”：当任务与工具集合扩展时，系统对规则工程的依赖可降低（Schick 等，2023）。([arXiv][4])

但在金融咨询场景，工具学习并不能替代合规与风险控制：工具输出的“事实正确”并不等价于“表达合规”。因此工具链更适合作为增强证据充分性与可引用性的手段，而**输出边界仍需独立合规模块进行过程约束**。这一点也与“系统化智能”框架强调的模块分工相一致（Karpas 等，2022；Wang 等，2023）。([arXiv][3])

### （四）从“短期上下文”到“长期认知状态”：记忆、反思与自我纠错机制

多轮对话智能体若要实现长期陪伴，仅靠上下文拼接难以维持稳定一致性，因此“显式记忆”逐渐成为主流架构组件，并进一步演化出“反思—记忆—再规划”的闭环。Shinn 等（2023）提出 Reflexion，用语言化反馈驱动自我反思：智能体将失败原因与改进策略写入情景记忆缓冲区，在后续尝试中通过“可读文本记忆”而非参数更新实现试错改进，从而提升连续任务执行能力（Shinn 等，2023）。([arXiv][5])
Park 等（2023）提出 Generative Agents 的“记忆流—反思—规划”三段式架构：将经历持续记录为自然语言记忆，通过反思生成更高层抽象的稳定认知，再在规划阶段动态检索相关记忆以驱动后续行为，从而支持更长时间跨度的一致行为生成（Park 等，2023）。([arXiv][6])

上述工作共同指向一项关键跃迁：记忆不再只是“召回相似文本”，而是用于维护**可持续更新的认知状态**并支持纠错与计划更新（Shinn 等，2023；Park 等，2023）。([arXiv][5]) 因此，你文中“短期优先、长期补充、画像约束”的分层记忆编排，可更明确地表述为：短期记忆保障语义连续与局部一致性，长期记忆提供跨时间背景与依据，用户画像以结构化状态变量形式贯穿生成全过程，以降低跨轮遗忘与偏好漂移带来的决策不一致风险（Wang 等，2023）。([arXiv][1])

### （五）从“文本世界”到“真实世界”：具身智能体与可执行性约束

当智能体从文本交互走向对环境的真实操作，架构核心挑战转向“计划的可执行性”。Ahn 等（2022）提出 SayCan，将语言模型的高层动作提议与低层技能的可供性/价值评估结合，用技能价值函数约束 LLM 的动作选择，使生成计划与环境能力对齐，避免“说得对但做不到”（Ahn 等，2022）。([arXiv][7])
Wang 等（2023）提出 Voyager，展示了更长期的开放式具身学习：通过自动课程生成、可复用技能库与迭代式提示机制，将环境反馈、执行错误与自我验证纳入循环，在无需微调模型参数的情况下实现技能累积与复用（Wang 等，2023）。([arXiv][8])

尽管证券咨询不属于物理具身，但同样存在“可执行性约束”：合规规则、风险披露清单、适当性要求等构成了金融场景的“制度环境”。类比而言，金融智能体也应将“可发布/可合规”作为显式约束引入生成过程，而不是生成后再被动筛查。你的合规管理模块可据此表述为对 SayCan“可执行性约束”思想的领域化实现：在生成阶段即补足风险提示并抑制禁区表达，从架构层面降低跨轮漂移带来的合规风险（Ahn 等，2022）。([arXiv][7])

### （六）综述视角下的统一框架：Profiling–Memory–Planning–Action

为系统化刻画智能体架构要素，Wang 等（2023）在综述中提出较为统一的分析框架，将智能体拆解为**画像/角色设定（profiling）、记忆（memory）、规划（planning）与行动（action/tool use）**等模块，并讨论其组合形态与评测策略（Wang 等，2023）。([arXiv][1]) 结合证券投资咨询场景的工程需求，该框架还可进一步领域化：一方面，“画像”需要以结构化形式承载风险等级、期限偏好与禁忌约束等关键状态；另一方面，“行动/工具”需要提供事实证据与可引用依据；同时必须引入独立“合规模块”对输出进行过程约束与可审计化封装。基于此，你可将长期一致性输出进一步拆解为“记忆增强、推理—行动闭环与合规管理”三类可工程化能力，并以接口化方式实现模块解耦，便于扩展并对接真实业务流程与监管要求。

### （七）小结：智能体架构演进的主线是“可控系统化”

综上，智能体架构的发展呈现清晰主线：从端到端生成走向“推理—行动”闭环（ReAct）（Yao 等，2022），从单一模型走向模块化系统（MRKL）（Karpas 等，2022），从工具可用走向工具可学习（Toolformer）（Schick 等，2023），从短期上下文走向长期认知状态与反思纠错（Reflexion、Generative Agents）（Shinn 等，2023；Park 等，2023），并在具身与开放环境中引入可执行性与反馈学习机制（SayCan、Voyager）（Ahn 等，2022；Wang 等，2023）。([arXiv][2])
这些工作共同指向一个结论：多轮对话智能体的核心竞争力不再是单纯“生成能力”，而是“状态管理 + 工具执行 + 过程约束”的系统能力（Wang 等，2023）。([arXiv][1]) 这也为你后续提出面向证券咨询的“记忆—工具—合规”协同架构提供理论与工程脉络支撑：通过分层记忆与用户画像维护长期一致性，通过工具链增强事实支撑与可引用性，通过合规模块实现风险披露与表达边界的稳定控制，从而将长期陪伴式投顾从“模型能力展示”推进到“可评测—可迭代—可交付”的工程闭环。
